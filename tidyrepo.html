Tidy Repo

A curated list of the best WordPress Plugins

I’ve worked with WordPress since about 2006. Throughout my time with the it, I’ve gotten a good lay of the land. As a newcomer, what really drew me to WordPress was it’s extendability. With actions, hooks and filters for virtually everything, it’s easy to create a modularized chunk of code that can be plugged into any WordPress install. We call these plugins. And there are over 20,000 of them in the WordPress.org repository. Somare are great. Others haven’t been updated in years and will crash your site. If you know what to look for, this is somewhat avoidable, but even so, it is a difficult landscape to navigate.

The solution seemed simple. A site that contains only the plugins that have been tested to work. Ideally, a place where you can find the best plugin for the job, and only plugins that are 100% functional and useful. I was hard pressed to find a site like that, so I created it.

Planning It Out

When embarking on a side project, I try my hardest to treat myself like a client. Before jumping into the site, I know I needed a list of requirements and a name. The name was easy, Tidy Repo (it came to me in my sleep). The requirements were a bit more involved.

  • A small and concise list of plugins, organized in useful categories.
  • Emphasis on search and navigation to make the right plugin findable.
  • Links to the download the plugin and related resources all in one place

These felt right to me, but there was a missing piece. When a user came to the site, I wanted them to be able to get started with the plugins right away. This meant that each plugin on the site should be accompanied by a tutorial for the average beginner. This meant a lot more work for me, but I couldn’t picture the site without it.

Working Out the Design

One of the biggest influences on Tidy Repo is Unheap, a similar repository of jQuery plugins. They’re grid structure lended itself to the project’s goals, but they also added another fun component. For each plugin that was added, they created a demonstrative graphic. This meant that searching and filtering became largely a visual experience and reduced cognitive load. I wanted to bring this into Tidy Repo, so when sketching out the design, I made sure the graphics were brought to the forefront.

But I also had that my own twist, the tutorials. It can be frustrating for a user to have to rummage through sidebars and auxiliary content to actually read something. So I settled on creating a single-column post page that was wholly focused on a pleasant reading experience.

And, of course, search and navigation would be extremely important. On the homepage, the search bar would be the first thing you see, followed by a list of categories, not tucked away, but right there, easily accessed.

Last up was the logo. I decided to pull from WordPress’ own icon library, Genericons, to bring some of their aesthetic in. Tidy Repo was to be a site to find plugins, so a “search” and “plugin” icon side by side seemed fitting.

Bringing it Into the Browser

When I finish sketching a site, I usually bring it into the browser immediately, to design there with HTML + CSS in a static prototype. In order to be able to move quickly, I used Harry Robert’s Inuit CSS library, which I had been meaning to try out. It uses SASS to create a solid development foundation upon which to create a design. That, plus my usual Grunt and Git one-two punch and I’m able to get designs off the ground pretty quickly.

As I worked through the design, new possibilities revealed themselves through the interactions and mechanics of the browser, as they tend to do. I try to embrace these where I can instead of fighting against them. And on my own project, I had no problem taking them as far as they could go. For instance, I create a slide-out button on each post page, so users could flip through plugin walkthroughs without having to go back to the homepage.

The biggest challenge was giving content the proper room to breathe, while still showing a sortable and expansive list. I tweaked margins, padding and typography quite a bit. Eventually, I found a good balance with ample space, larger then average type size on posts, and slightly smaller type on the index page.

Responsive First

While working out the details of Tidy Repo, keeping it responsive was always top of mind. It has sort of become my default. Whenever I create a new feature, I squish the screen to see how it looks. Then, I can quickly iterate on it until it fits a variety of different screen sizes. Only then do I move on. This certainly isn’t a mobile-first strategy, which I’ve used in the past, but I do try to prioritize content over any particular screen size. Only when it is absolutely necessary do I make any substantial changes at different sizes, for instance the navigation menu on Tidy Repo.

Finishing It Up in WordPress

Since Tidy Repo is all about WordPress, I kind of had to put my money where my mouth is and use the platform. That really was no problem, since I’ve used WordPress dozens of times in the past. For a content-heavy site like the one I was created, I knew it was the perfect thing.

When moving from the front-end to the back-end, I used my HTML prototype as a foundation, swapping out template tags or database queries where it’s necessary. The only addition I had to make was the use of Advanced Custom Fields for some extra metadata and Disqus for comments. Other then that, the transition was smooth and fairly quick.

All in all, I went from having the idea to create Tidy Repo to actually launching it in about a month. Every Monday, Wednesday and Friday I post a new plugin, with my own walkthrough and a header graphic of my design. I’ve been fortunate enough to be taken in by many members of the WordPress community, and I get new feedback and plugin suggestions basically every day.

Of all the side projects I’ve worked on, Tidy Repo has been and continues to be be the greatest effort, but also the most rewarding. I don’t see it going away anytime soon.
